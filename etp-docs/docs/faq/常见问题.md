
## 1、Linux二进制安装运行`etps/etpc`时出现类似`GNU libc2.32`错误解决方案


:::tip 提示

`etp`运行环境要求`Linux`系统中`GNU libc`版本为`2.34`及以上版本

:::

解决方案：

- 升级`GNU libc` 到`2.34`
- 采用`高版本`的Linux服务器

## 2、`docker`安装`etp`，`etpc`客户端正常连接`etps`服务端，但是无法通过代理端口（公网端口）连接到内网指定服务

- 确保端口号在`云服务器`防火墙中已开放。
- 确保在`启动容器`时将`代理端口`暴露出来，建议一次性暴露常用端口。
